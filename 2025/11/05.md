## わかったこと

### シリアライザとは

シリアライザ（serializer）は、データを保存・送受信できる形に変換する仕組みのこと。Django では、セッションや API 通信などで使われる。

---

#### シリアライズとデシリアライズ

- シリアライズ（serialize）：Python のデータ → 文字列（保存・送信用に変換）
- デシリアライズ（deserialize）：文字列 → Python のデータ（元に戻す）

---

#### Django セッションでの使われ方

Django のセッションは内部的に Python の辞書で扱う。保存時に「辞書 → 文字列」へシリアライズし、読み出し時に「文字列 → 辞書」へデシリアライズする。

例（概念）:
    保存時: {"user_id": 5, "theme": "dark"} → '{"user_id": 5, "theme": "dark"}'
    読出時: '{"user_id": 5, "theme": "dark"}' → {"user_id": 5, "theme": "dark"}

---

#### Django の設定

settings.py の例（説明のみ。実際のコードは下のコメントを外して使用）:
    # SESSION_ENGINE = "django.contrib.sessions.backends.db"
    # SESSION_SERIALIZER = "django.contrib.sessions.serializers.JSONSerializer"

- SESSION_ENGINE は保存先（ここでは DB セッション）
- SESSION_SERIALIZER は変換方式（JSONSerializer が標準かつ推奨）

---

#### なぜ JSONSerializer なのか

- 安全で可読性・互換性が高い
- Django の JSON エンコーダにより date/datetime/Decimal なども扱える
- 旧来の PickleSerializer は任意コード実行の危険があるため非推奨

---

#### セッションに入れてよい／避けるべきデータ

入れてよい（JSON で表現できるもの）:
- str, int, float, bool
- dict, list
- date, datetime, Decimal（Django のエンコーダ経由で対応）

避けるべき:
- Model インスタンスや QuerySet（主キーなど最小情報にする）
- 巨大オブジェクト、バイナリ

例（NG/OK の考え方）:
    NG: request.session["user"] = user_obj
    OK: request.session["user_id"] = user.id
