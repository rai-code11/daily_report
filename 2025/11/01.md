### PythonとJavaScriptにおけるインスタンスと継承の違い

PythonとJavaScriptのどちらも「設計（クラスや関数）から生成された実体」をインスタンスと呼ぶ点は共通している。  
ただし、インスタンス生成とメソッド探索の仕組みが根本的に異なる。

### Pythonの場合
- クラスをもとにインスタンスを生成する  
- メソッドは `__class__` から親クラスへと順に探索される（メソッド解決順序: MRO）  
- すべてのクラスは最終的に `object` クラスを基底として持つ  
- クラス継承は静的（定義時に継承構造が決まる）

```python
class A:
    def greet(self):
        print("Hello from A")

class B(A):
    pass

b = B()
b.greet()  # => Hello from A
```

### JavaScriptの場合
- コンストラクタ関数やオブジェクトリテラルからインスタンスを生成する  
- メソッドは `__proto__` をたどって `prototype` チェーン上で探索される  
- すべてのオブジェクトは最終的に `Object` を基底とする  
- 継承は動的（実行時にprototypeチェーンを変更できる）

```javascript
function A() {}
A.prototype.greet = function() {
    console.log("Hello from A");
};

const b = new A();
b.greet();  // => Hello from A
```

まとめると、インスタンスという概念は共通だが  
Pythonは静的なクラス継承、JavaScriptは動的なプロトタイプ継承という違いがある。

### Object.hasOwn と in 演算子の違い

### Object.hasOwn()
- オブジェクト自身がそのプロパティを持っているかを判定する  
- 継承元（prototype）のプロパティは対象外  

### in 演算子
- オブジェクト自身にプロパティがなくても  
  prototype チェーンをたどって存在を確認する  

### 例

```javascript
const obj = {};

// obj自身にはtoStringが定義されていない
console.log(Object.hasOwn(obj, "toString")); // false

// prototype（Object.prototype）まで探索するためtrueになる
console.log("toString" in obj); // true
```
