## わかったこと
```bash
heroku create 新アプリ名 → 新しいHerokuアプリが1個できる（URLも発行される）
heroku git:remote -a 新アプリ名 → 今のローカルリポジトリと、その新アプリが heroku リモートとして紐付く
heroku addons:create heroku-postgresql:essential-0 -a 新アプリ名 → その新アプリ専用の Postgres(DB) が作られて、DATABASE_URL がConfig Varsに入る
heroku config:set SECRET_KEY='ランダムな長い文字列' -a x-clone
heroku config:set DJANGO_SETTINGS_MODULE='config.settings.production' -a x-clone
heroku config -a x-clone
```
* DATABASE_URL
* SECRET_KEY
* DJANGO_SETTINGS_MODULE
の3つが並んでたらOK

- flake8をパスするとは
ターミナルで flake8 . を打つ
エラーや警告が表示される →　修正箇所を教えてくれる
その指摘を見て コードを修正する
もう一回 flake8 . を実行する
何も表示されなくなったらflake8をpassした状態

- Python は 隣り合う文字列リテラルを自動で結合してくれる （カンマ , や + が入ってなければOK）
だから 2 行に分けても、実際の中身は 1 つの長い文字列になる

- データとして絶対NGなもの → モデル／DB
画面ごとの入力ルールや優しさ → フォーム
