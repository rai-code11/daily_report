# PythonとJavaScriptの関数・構文の違いまとめ

## 関数の引数の扱い

### Python
- 引数の扱いが厳格  
- デフォルト値のない引数は、関数呼び出し時にすべて渡す必要がある  
- 引数の数が不足または超過していると `TypeError` が発生する  
- 定義された数と名前が一致しなければ実行できない  

```python
def add(a, b):
    return a + b

add(1)        # => TypeError（引数不足）
add(1, 2, 3)  # => TypeError（引数超過）
```

### JavaScript
- 引数の扱いが柔軟  
- 引数が不足していてもエラーにならず、足りない部分は自動的に `undefined` になる  
- 逆に引数が多くてもエラーにならず、余分な引数は無視される  
- 引数の数よりも「渡す順番」が重要  

```javascript
function add(a, b) {
    console.log(a, b);
}

add(1);         // a=1, b=undefined
add(1, 2, 3);   // a=1, b=2（3は無視される）
```

## do-while文

`do-while` 文は `while` 文と似ているが、条件判定のタイミングが異なる。  
必ず一度は処理を実行してから条件を確認する。

```javascript
do {
    実行する文;
} while (条件式);
```

実行フロー:
1. 実行する文を処理  
2. 条件式を評価し、`true` なら再び実行、`false` なら終了  

```javascript
const x = 1000;
do {
    console.log(x); // => 1000
} while (x < 10);
```

この場合、条件を満たしていなくても最初の1回は実行される。

## 分割代入（Destructuring assignment）

オブジェクトから特定のプロパティをまとめて取り出す構文。  
左辺にオブジェクトリテラルのような構文で変数名を定義し、右辺の対応するプロパティを代入する。

```javascript
const languages = {
    ja: "日本語",
    en: "英語"
};

const { ja, en } = languages;

console.log(ja); // => "日本語"
console.log(en); // => "英語"
```

従来の方法では、次のように2行で書いていたものを1行にまとめられる。

```javascript
const ja = languages.ja;
const en = languages.en;
```

## Object.freezeによるオブジェクトの凍結

`Object.freeze()` を使うとオブジェクトを凍結し、プロパティの追加や変更を禁止できる。  
凍結後に変更しようとするとエラーが発生する。

```javascript
"use strict";

const object = Object.freeze({ key: "value" });
object.key = "newValue"; // => TypeError: "key" is read-only
```

ポイント:
- `Object.freeze()` はオブジェクトの内容を固定する  
- `strict mode` と組み合わせて使う必要がある  
- `strict mode` でない場合、変更は単に無視されて例外は発生しない

## まとめ

- Python は引数チェックが厳密で、数や名前が一致しなければエラーになる  
- JavaScript は引数に柔軟で、足りない部分は `undefined`、余分な引数は無視される  
- `do-while` 文は、条件に関係なく最初の1回は必ず実行される  
- 分割代入でオブジェクトのプロパティを簡潔に取り出せる  
- `Object.freeze()` を使うとオブジェクトの変更を防ぎ、`strict mode` 下で安全に固定できる
