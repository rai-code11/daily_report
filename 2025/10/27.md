## わかったこと
### Heroku で Django シェルを使ってデータを削除する手順

### 実行コマンドと流れ

```bash
# Heroku 上で Django の対話シェルを起動
heroku run python manage.py shell
```

```python
# モデルをインポート
from product_list.models import ProductList

# 全レコードを削除
ProductList.objects.all().delete()
```

---

### 処理の意味

1. **`heroku run python manage.py shell`**  
   - Heroku のコンテナ内で Django のシェル（Python 対話モード）を起動するコマンド。  
   - ローカル環境ではなく、**Heroku 本番環境のデータベース**に直接アクセスする。

2. **`from product_list.models import ProductList`**  
   - Django アプリ `product_list` に定義されている `ProductList` モデルをインポートする。  
   - このモデルはデータベース上の `product_list` テーブルに対応。

3. **`ProductList.objects.all().delete()`**  
   - `ProductList` テーブル内のすべてのレコードを削除する。  
   - 戻り値は `(削除件数, {'app.ModelName': 件数})` の形式で表示される。

---

### 注意点

- この操作は **Heroku の本番データを直接削除する**。  
  → テスト目的以外では実行しないように注意。  
- 特定のレコードだけ削除したい場合は、フィルタを追加して制限すること。  
  例:  
  ```python
  ProductList.objects.filter(id=1).delete()
  ```
- 誤って削除した場合、Heroku Postgres のバックアップ機能がない限り復元は不可。
