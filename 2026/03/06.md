- State更新関数には Dispatch<SetStateAction<型>> を使う
```

```
TypeScriptはデフォルトでは、「Userオブジェクトの中身を、動的なキー（変数 prop）でいきなり覗き見ること」**を禁止している。なぜなら、その prop の中身が age（数値）なのか name（文字列）なのか、あるいは存在しないプロパティなのかが不透明だから。
解決方法：keyof User を使う
引数の prop に、「これは必ず User 型が持っているキーのどれかだよ」と教えてあげるのが一番スマート。
```TypeScript

// useDisplay.ts の中

// 修正前: const handleSort = (prop, direction) => {
// 修正後: prop の型を keyof User に限定する
const handleSort = (prop: keyof User, direction: "asc" | "desc") => {
  const sortedList = [...displayUserList].sort((a, b) => {
    
    // a[prop] を取得（User型に存在するキーなのでアクセス可能になる）
    const valA = a[prop];
    const valB = b[prop];

    // 数値同士の比較であることを明示的にチェック（または型アサーション）
    if (typeof valA === "number" && typeof valB === "number") {
      return direction === "asc" ? valA - valB : valB - valA;
    }
    
    // 文字列などの比較が必要な場合はここに追加
    return 0;
  });
  setDisplayedUserList(sortedList);
};

```

- Reactの onChange イベントを正しく扱うにはジェネリクス <...> の中には、そのイベントが発生するHTML要素を入れる
```TypeScript
const onChangeNewUser = (event: React.ChangeEvent<HTMLInputElement>) => { const { name, value } = event.target; setNewUser({ ...newUser, [name]: value }); };
```
- プルダウン（select）など他の要素でもこの関数を使う場合は、以下のように「または（|）」で繋ぐ。
```TypeScript

const onChangeNewUser = (
  event: React.ChangeEvent<HTMLInputElement | HTMLSelectElement>
) => {
  const { name, value } = event.target;
  setNewUser({ ...newUser, [name]: value });
};
```
