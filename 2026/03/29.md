## わかったこと

- 機能	役割	使う場所	Prismaの操作例
- Server Components	取得 (Read)	page.tsx など	prisma.todo.findMany()
- Server Actions	追加 (Create)	actions.ts など	prisma.todo.create()
- Server Actions	更新 (Update)	actions.ts など	prisma.todo.update()
- Server Actions	削除 (Delete)	actions.ts など	prisma.todo.delete()



- サーバーとクライアントの使い分け表
- 特徴	Server Components (標準)	Client Components ("use client")
- 主な役割	データの取得、SEO対策、静的な表示	インタラクティブな操作、リアルタイムな反応
- 実行場所	サーバー側（Node.js環境）	ブラウザ側（ユーザーのPC）
- Prisma	直接使える	直接使えない（セキュリティ上NG）
- Hooks	useStateなどは使えない	useState, useEffectなどが使える
