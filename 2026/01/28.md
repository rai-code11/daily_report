三項演算子
```js
function getFee(isMember) {
  return isMember ? "$2.00" : "$10.00";
}

console.log(getFee(true));
// 予想される結果: "$2.00"

console.log(getFee(false));
// 予想される結果: "$10.00"

console.log(getFee(null));
// 予想される結果: "$10.00"
```

- Reactは「DOM操作」を書くのではなくUIの状態と振る舞いを書くので非常に簡単
```js
const [value, setValue] = useState(null);

value：今の状態（stateの値）

setValue：その状態を更新するための関数

const [value, setValue] = useState(null);
```
- これは useState(null) の戻り値を分解してる。

- useState は内部的にだいたいこういうものを返すイメージ：

```js
[value(現在の値), setValue(値を変えるための関数)]
つまり useState(null) は（概念的に）こんな配列を返してる：

[null, function(新しい値) { /* Reactが状態を更新する */ }]
だから、

value には「今の状態の値」が入る

setValue には「状態を更新するための関数」が入る

setValue('X');
```

state の値を更新する

- そのコンポーネントをもう一回実行する
