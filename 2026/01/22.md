## わかったこと

1. ボタンが「変身」していく流れ
ボタンが押されるたびに、プログラムは「今のこのボタンの役割は何だっけ？」と確認
1. 最初: ボタンは「編集」
2. 1回目のクリック:
    * if 文が「あ、今は『編集』だね」と判定。
    * 処理の中で文字を**「保存」**に書き換える。
    * 画面が入力欄に変わる。
3. 2回目のクリック:
    * if 文が「今は『編集』じゃない（『保存』になってる）ぞ」と判定。
    * else の処理が走り、文字を**「編集」**に戻す。
    * 画面がテキスト（pタグ）に戻る。
2. もし if 文がなかったら？
もし if 文を使わずに「入力欄に変える処理」だけを書いてしまうと
* 1回押す：入力欄に変わる。
* 2回押す：さらに入力欄を作ろうとして、エラーになったり、入力欄の中にさらに入力欄ができたりする
