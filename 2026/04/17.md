## わかったこと
リモートリポジトリのブランチをローカルに持ってきて、ブランチを切ってブランチを切り替えるコマンド
```
git switch -c ローカルに作成するブランチ名 リモートブランチ名
```

- Reducer	リデューサー	注文を受けてデータを書き換える「管理人」
- Store	ストア	データを溜めておく「倉庫」
- Action	アクション	「〇〇して！」という「注文書」
- Dispatch	ディスパッチ	注文書を「送り出す」こと

**Reduxという神機能**
- 基本的にReact単体でアプリを作成するとコンポーネントからコンポーネントへ情報を渡すというリレーが発生するが、ReduxはStoreというデータ倉庫を作り、そこからuseSelectorでデータを直接取得するため非常にわかりやすいし、どこからでもデータを持ってくることができる。

**コミットメッセージの修正**
- 直前のメッセージミスは git commit --amend で上書きできる
