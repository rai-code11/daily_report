## 全体の構造（3層）                                                                                   
                  
  今回の実装は3つのファイルに役割が分かれている                                                     
                                                                                                         
  ```                                                                                                    
  atoms/Popover/index.tsx        ← 第1層: Radixの薄いラッパー（共通部品）
          ↓ 使われる                                                                                     
  molecules/PopupScore/index.tsx ← 第2層: ポップオーバーの中身                                           
          ↓ 使われる                                                                                     
  organisms/Header/index.tsx     ← 第3層: 実際にポップオーバーを開く場所                                 
  ```                                                                                                    
                  
  ---

  ## 第1層：`atoms/Popover/index.tsx`                                                                    
   
  Radix                                                                                                  
  UIのPopoverをそのまま使うのではなく、このプロジェクト用に**薄くラップ（包んで）**した共通部品。
                                                                                                         
  ```tsx                                                                                                 
  import * as PopoverPrimitive from "@radix-ui/react-popover";
  ```                                                                                                    
                  
  Radix UIの本体を `PopoverPrimitive` という名前で読み込んでる。                                     
   
  ### エクスポートされている5つのパーツ                                                                  
                  
  ```                                                                                                    
  Popover         → ポップオーバー全体の管理（開閉状態を持つ）
  PopoverTrigger  → クリックすると開くトリガー
  PopoverContent  → 表示される吹き出しの中身の枠
  PopoverAnchor   → 位置の基準点（今回は未使用）
  PopoverArrow    → 吹き出しの△（矢印）
  ```                                                                                                    
   
  ### `PopoverContent` の `variant` という工夫                                                           
                  
  ```
  variant = "default"  → 白背景・影・パディング付き（標準スタイル）
  variant = "unstyled" → スタイルなし（自分で全部決められる）
  ```                                                                                                    
   
  今回のスコアポップオーバーはデザインが特殊（濃い青背景）なので `unstyled` を選んでる。             
                  
  ### `PopoverArrow` のカスタマイズ                                                                      
                  
  標準のRadixの矢印はSVGパスで描画されますが、今回はデザイン通りの見た目にするために**SVG画像ファイル**（
  `/assets/icon/popover-arrow.svg`）に差し替えている。
                                                                                                         
  ```tsx          
  if (src) {
    return (
      <PopoverPrimitive.Arrow asChild>
        <Image src={src} ... />  ← Radixの矢印をNext.jsのImageに差し替え
      </PopoverPrimitive.Arrow>                                                                          
    );
  }                                                                                                      
  ```

  ## 第2層：`molecules/PopupScore/index.tsx`

  ポップオーバーの**中身だけ**を担当するコンポーネント。ポップオーバーを開く仕組みは知らなくていい。 
   
  ```tsx                                                                                                 
  export interface PopupScoreProps {
    isViewScore: boolean;          // 今スコアが表示中かどうか
    onToggleViewScore: () => void; // トグルされたときに呼ぶ関数                                         
  }
  ```                                                                                                    
                  
  外から「今の状態」と「切り替えたいときに呼ぶ関数」を受け取るだけで、自分では状態を持たない。         
   
  ```                                                                                                    
  ┌────────────────────────────────┐
  │  スコアを表示する    [スイッチ] │  ← isViewScore + onToggleViewScore
  │────────────────────────────────│                                                                     
  │  ホームの動画を自動再生する [■]│  ← 現在は固定（未実装）
  └────────────────────────────────┘                                                                     
  ```             
                                                                                                         
  ### スタイル（`index.css.ts`）

  ```
  container → 濃い青背景・角丸・幅281px
  row       → 横並び・両端揃え
  label     → グレーのテキスト・14px                                                                     
  ```

  ## 第3層：`organisms/Header/index.tsx`

  実際にポップオーバーを組み立てて使っている場所。

  ```tsx
  // Headerが受け取るprops（外から渡される）
  onSwitching: () => void;  // スコア切り替え関数                                                        
  isViewScore: boolean;     // スコア表示状態
  ```                                                                                                    
                  
  ### ポップオーバーの組み立て                                                                           
   
  ```tsx                                                                                                 
  <Popover>       
    {/* トリガー: 目のアイコンボタン */}
    <PopoverTrigger asChild>
      <button type="button">
        <Icon svgName="eye-off" ... />
      </button>                                                                                          
    </PopoverTrigger>
                                                                                                         
    {/* 吹き出しの中身 */}
    <PopoverContent
      variant="unstyled"      // 独自デザインのためスタイルリセット
      align="end"             // 右端揃え                                                                
      side="bottom"           // アイコンの下に表示
      sideOffset={1}          // アイコンとの縦の隙間                                                    
      alignOffset={-27}       // 右に27px微調整                                                          
      avoidCollisions={false} // 画面端での自動回避をOFF
    >                                                                                                    
      <PopoverArrow
        src="/assets/icon/popover-arrow.svg"
        className="rotate-180"  // SVGが上向きなので180度回転して下向きにする                            
        width={13.86}
        height={12}                                                                                      
      />          
      <PopupScore
        isViewScore={isViewScore}
        onToggleViewScore={onSwitching}
      />                                                                                                 
    </PopoverContent>
  </Popover>                                                                                             
  ```             

  ### 位置調整のオプション詳解

  ```
  align="end"             → ポップオーバーをトリガーの右端に合わせる
  side="bottom"           → トリガーの下側に表示する
  sideOffset={1}          → トリガーとの縦の距離（1px）
  alignOffset={-27}       → 右方向に27pxずらす（矢印位置の微調整）
  avoidCollisions={false} → 画面端に近くても自動でずれないようにする                                     
  ```
                                                                                                         
  ---             

  ## データの流れ

  ```
  どこかのページ（親）
      ↓ isViewScore, onSwitching を渡す
  Header
      ↓ そのまま渡す
  PopupScore
      ↓ Switch コンポーネントに渡す
  Switch（トグルボタン）                                                                                 
      ↑ ユーザーがON/OFFすると onToggleViewScore() が発火
      ↑ → Header の onSwitching() → ページの状態が更新                                                   
  ```             

  **PopupScore 自身は状態を持っていない**のがポイント。状態は上の親が持っていて、PopupScoreは「表示と通知
  」だけを担当しています。これを**「状態の持ち上げ（Lifting State Up）」**と言う

  ---

  ## PopupCalendarとの比較

  今回のスコアポップオーバーは既存の `PopupCalendar` と同じ仕組みを使ってる。                        
   
  | | PopupCalendar | PopupScore |                                                                       
  |---|---|---|   
  | 開閉状態 | 自分で `useState` で管理 | Radixが自動管理 |
  | Arrow | なし | SVG画像を使用 |                                                                       
  | スタイル | default（白背景） | unstyled（独自デザイン）|
  | 中身 | カレンダー | スコア切り替えスイッチ |                                                         
                                                                                                         
  `PopupCalendar` は自分で `open/onOpenChange` を持っていますが、`PopupScore`                            
  はRadixに任せているのでよりシンプル

  ## 全体の流れ

  ```
  ユーザーが目のアイコンをクリック
          ↓
  Radix が Popover を開く（状態管理は自動）
          ↓                                                                                              
  PopoverContent が画面に現れる
    ┌──────────────────────────┐                                                                         
    △  ← SVGの矢印（rotate-180）
    │  スコアを表示する  [ON]  │
    │  ホームの動画...   [OFF] │                                                                         
    └──────────────────────────┘
          ↓                                                                                              
  スイッチをON/OFFする
          ↓
  onToggleViewScore() 発火 → 親の状態が更新 → isViewScore が変わる → スコア非表示
  ```
