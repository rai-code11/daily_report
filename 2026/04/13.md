## わかったこと

### 1. Claude Code 基本コマンド・機能
CLI上で動作するAIエンジニアリングツール「Claude Code」の主要な操作と設定について。

| コマンド / キーワード | 内容・役割 |
| :--- | :--- |
| `/init` | **プロジェクトの概要作成**。リポジトリの構造や目的をスキャンし、AIが文脈を理解するための初期化を行う。 |
| `/compact` | **コンテキストの要約**。会話が長くなった際、これまでの内容を要約して履歴を整理し、トークン消費を節約する。 |
| `/clear` | **セッションのリセット**。現在の会話履歴を消去し、別のタスクに切り替える際に使用する。 |
| `/agents` | **エージェント機能**。特定のガイドラインやドキュメントを元に、高度な分析や提案を受ける。 |

### 2. 設定とドキュメント参照
* **`settings.json` の場所**:
    * Claude Code の動作設定を定義するファイル。公式ドキュメント（`claude-code-guide`）を介して、プロジェクトごとの最適な配置場所や構成を確認済み。
* **エージェント活用**:
    * `@“claude-code-guide(agent)”` を指定して質問することで、最新の公式ドキュメントに基づいた正確な情報を抽出可能。

### 3. フロントエンド・開発環境ツール
プロジェクトの質を高めるために確認したライブラリおよびツール群。

* **React Rewards**:
    * ボタンクリック時などに「紙吹雪（Confetti）」などのアニメーションエフェクトを簡単に実装できるライブラリ。
* **shadcn/ui**:
    * コピー＆ペーストで利用可能な高品質なUIコンポーネントライブラリ。Next.js環境での開発効率を大幅に向上させる。
* **ESLint**:
    * 静的コード解析ツール。コードの構文エラーや、プロジェクト固有のコーディング規約（TypeScriptの型定義など）を自動でチェックする。
 

### その他
* **開発フロー**: 新しい環境では `Cursor` と `Claude Code` を併用し、効率的なコーディングスタイルを構築中。
* **トークン管理**: 長時間の作業では `/compact` を活用し、常に最適なレスポンス精度を維持するのがコツ。
